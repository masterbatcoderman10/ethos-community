## Overview

Kushian™ is a Connection-as-a-Service application powered by Ethos Community™ and built for the IsDB Group Innovation and Startups Pitch Competition. The visual identity fuses **development-finance gravitas** with **editorial warmth** — a palette of cream paper, deep forest ink, and restrained gold accents that signals trust, transparency, and institutional credibility without feeling cold or corporate.

The design language is intentionally warm and dignified. Serif display typography (three interchangeable pairings) anchors the editorial tone, while a monospace label system (JetBrains Mono) provides the precision and traceability expected of finance and audit contexts. Every surface — from beneficiary profiles to impact dashboards — shares the same restrained color logic and generous whitespace.

## Colors

The palette is built on a warm paper/cream ground with a single dominant green and a single gold accent. Red exists only for functional urgency states.

- **Cream (#f7f4ee)** — Primary background. Warm, slightly desaturated paper tone that avoids the clinical feel of pure white.
- **Cream-2 (#efeae0)** — Secondary background for strips, cards on cream, and subtle elevation.
- **Ink (#141512)** — Primary text. Near-black with a slight warm olive bias.
- **Ink-soft (#3a3a36)** — Secondary text, descriptions, lede paragraphs.
- **Muted (#7a7568)** — Tertiary text, meta labels, disabled states.
- **Line (#e2dccf)** — Borders, dividers, hairlines. Warm grey derived from the cream family.
- **Green (#0e3b2e)** — Primary action color. Deep forest green used for buttons, active nav, verification badges, and dark panels. Signals growth, trust, and Islamic finance compatibility.
- **Green-2 (#0a2a21)** — Darker green for hover states, footer background, and ink-dark panels.
- **Gold (#c9a961)** — Accent. Used sparingly for eyebrow labels, stat highlights, featured card backgrounds, and CTA band buttons. The sole "warmth" injection.
- **Gold-2 (#a8884a)** — Muted gold for secondary labels and eyebrow text.
- **Red (#a13628)** — Urgency, alerts, downward deltas. Used only in urgency bars and urgent tags.
- **Red-2 (#7a2519)** — Darker red for red hover states.
- **Blue (#1c3a5c)** — Reserved for the legal vertical iconography. Defined in CSS but not currently used as a UI accent.

### Hardcoded values in use

These values appear in the codebase but are not yet promoted to tokens:
- **Card hover border**: `#d4caac` — a slightly darker warm grey used on `.card:hover`.
- **Generic gold hover**: `#d8b86c` — lighter gold for `.btn-gold:hover` on light backgrounds.
- **Urgency tint**: `rgba(196, 98, 67, 0.08)` — faint red wash behind urgency bars.

### Color strategy

Restrained Committed palette: ~3 hues (cream/green/gold) with red as a functional signal. No gradients. No glassmorphism. No side-stripe decorative borders. Dark surfaces use solid fills (green-2 or ink), never translucent overlays.

The nav and demo tag use `backdrop-filter: blur()` for functional frosted-glass legibility over scrolling content. This is a UI necessity, not decorative glassmorphism.

## Typography

Three interchangeable font pairings, switchable at runtime via `body[data-pair]`:

1. **Newsreader + Manrope** (default) — High editorial contrast. Newsreader's sharp serifs and optical size axis give display text a newspaper gravitas. Manrope provides clean, warm geometric sans for body.
2. **DM Serif Display + DM Sans** — Slightly softer, more rounded serifs. DM Sans has a friendlier, more approachable rhythm.
3. **Instrument Serif + Inter Tight** — Tightest, most modern pairing. Instrument Serif has an italic with strong personality; Inter Tight is compact and contemporary.

All pairings share:
- Display weight: **400** (never bold — hierarchy through size and space, not weight)
- Display tracking: **-0.01em to -0.02em** (tight, editorial)
- Body weight: **400** regular

### Type roles

- **Display / h1-h3**: `var(--ff-display)`, weight 400, tight tracking. Sizes scale with `clamp()` for fluid responsiveness.
- **Body**: `var(--ff-body)`, weight 400, comfortable line-height (1.55–1.7).
- **Labels / meta / numbers**: JetBrains Mono, uppercase, wide letter-spacing (0.08em–0.18em), tiny sizes (10.5px–11px). This is the "audit trail" voice — every statistic, date, tag, and footer link uses it.
- **Blockquotes**: Display serif at 24–34px with a gold opening quotation mark.

## Elevation & Depth

No drop shadows on flat surfaces. Elevation is used sparingly and meaningfully:

- **Photo cards**: deep shadow (`0 30px 80px -30px rgba(20,21,18,.4)`) to lift editorial imagery off the cream background.
- **Card hover**: subtle shadow + border-color shift. Never default state.
- **Button hover**: lift (`translateY(-1px)`) + directional shadow.
- **Sticky nav**: `backdrop-filter: blur(14px)` with semi-transparent cream background.
- **Toast + demo tag**: high z-index (60/30), no blur on the element itself but tag uses `backdrop-filter: blur(8px)`.

## Shapes

- **Pill**: 999px border-radius — buttons, tags, pills, toasts, demo tag.
- **Small**: 4px — default photos, form fields, nav hamburger hover.
- **Medium**: 6px — cards, panels, avatars (xl), doc items, timeline containers, profile photos.
- **Large**: 8px — workflow bands, audit strips, dark feature blocks.
- **Circle**: 50% — avatars, logo mark, seal.

## Accessibility

- **Focus states**: All interactive elements (buttons, links, cards, chips, fields) must show a visible focus indicator. Default: `outline: 2px solid var(--green)` with `outline-offset: 2px–4px`.
- **Motion**: The reveal-on-scroll animation and counter animation should respect `prefers-reduced-motion: reduce`. In that mode, reveals should appear instantly (opacity 1, transform none) and counters should jump to final value.
- **Toast notifications**: Use `role="status"` and `aria-live="polite"` so screen readers announce them without stealing focus.
- **Color contrast**: Green (#0e3b2e) on cream (#f7f4ee) exceeds WCAG AAA. Gold (#c9a961) on green-2 (#0a2a21) exceeds WCAG AA. Red (#a13628) on cream-2 exceeds WCAG AA.
- **Touch targets**: Minimum 44px × 44px on mobile for all buttons, chips, persona pills, and nav items.

## Do's and Don'ts

### Do
- Use the cream background as the default canvas for every page.
- Let display type breathe — large sizes with tight tracking and generous line-height.
- Use JetBrains Mono for every label, stat, date, and meta element. The contrast between serif display and monospace meta is the signature rhythm.
- Keep the green as the single action color. Use gold only for emphasis, featured states, and dark-background CTAs.
- Use the hatch pattern (`repeating-linear-gradient`) for all photo placeholders — it signals "content loading" while staying on-brand.
- Maintain 120px section padding on desktop; let whitespace do the work of hierarchy.
- Provide visible focus indicators on all interactive elements.
- Respect `prefers-reduced-motion` for scroll reveals and counters.

### Don't
- Don't use bold weights (700) for display text. The system relies on size and space, not weight.
- Don't introduce new accent colors beyond green, gold, and functional red.
- Don't use gradients for text, backgrounds, or decorative elements.
- Don't use glassmorphism or translucent overlays on text.
- Don't use emojis or generic icon libraries — the system uses bespoke inline SVG line icons with 1.4px stroke.
- Don't shrink touch targets below 44px on mobile.
- Don't use side-stripe decorative borders or hero metric templates — these read as AI slop and break the editorial tone.
